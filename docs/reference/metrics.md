# Metrics reference

All Prometheus metrics exposed by Aitra Meter. Use these to build Grafana dashboards, Prometheus alerting rules, and KEDA ScaledObjects.

Metrics are exposed at:
- Measurement agent: `:9090/metrics` (per node)
- Aggregation service: `:8080/metrics`

---

## Measurement agent metrics

### `aitra_gpu_energy_joules_total`

**Type:** Counter  
**Source:** Measurement agent  
**Description:** Cumulative GPU energy consumed in joules since agent start.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `gpu_id` | GPU device ID (e.g. `0`, `1`) |
| `model_name` | Name of the model being served |
| `provider` | Energy provider name (e.g. `zeus`, `nvml`) |

---

### `aitra_gpu_power_watts`

**Type:** Gauge  
**Source:** Measurement agent  
**Description:** Current GPU power draw in watts.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `gpu_id` | GPU device ID |

---

### `aitra_gpu_idle_power_watts`

**Type:** Gauge  
**Source:** Measurement agent  
**Description:** GPU power draw in watts when `vllm:num_requests_running = 0`. Updated continuously during idle periods.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `gpu_id` | GPU device ID |

---

### `aitra_measurement_cv`

**Type:** Gauge  
**Source:** Measurement agent  
**Description:** Rolling coefficient of variation of J/token over the last 100 measurement windows. Values above 0.03 trigger the `unstable` flag.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `model_name` | Model name |

---

### `aitra_measurement_window_stable`

**Type:** Gauge  
**Source:** Measurement agent  
**Description:** `1` if the last measurement window passed the CV gate (CV < 3%), `0` if flagged unstable.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `model_name` | Model name |

## Aggregation service metrics

### `aitra_j_per_token`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Joules per output token for the current measurement window. The primary Aitra Meter metric. **This is accelerator (GPU) energy only** — it does not include host energy (CPU package, DRAM, board). For whole-node energy per token, see [`aitra_system_j_per_token`](#aitra_system_j_per_token), which is emitted when a host-energy provider is configured. The definition of this metric is unchanged: anyone with a baseline keeps it.

| Label | Description |
|---|---|
| `namespace` | Kubernetes namespace |
| `workload` | Workload type: `chat`, `code`, `reasoning`, `batch`, `unknown` |
| `model` | Model name |
| `hardware` | GPU tier from node label (e.g. `h100`, `l40s`) |
| `precision` | Precision from pod annotation: `fp16`, `fp8`, `bf16`, `unknown` |
| `calibration_tier` | `aitra_benchmark`, `reference`, `self_calibrated`, `uncalibrated` |
| `attribution_method` | `direct` or `proportional` |

---

### `aitra_cluster_j_per_token`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Cluster-wide J/token computed as Σ node energy ÷ Σ output tokens across all active workloads. Not an average of individual series.

| Label | Description |
|---|---|
| `cluster` | Cluster name (from `SiteConfig`) |
| `calibration_tier` | Dominant calibration tier for the cluster |

---

### `aitra_co2_per_token_grams`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Grams of CO₂ equivalent per output token. Derived: `J/token × gCO₂/kWh ÷ 3,600,000`.

| Label | Description |
|---|---|
| `namespace` | Kubernetes namespace |
| `workload` | Workload type |
| `model` | Model name |
| `hardware` | GPU tier |
| `carbon_source` | `electricitymaps`, `watttime`, `last_known`, or `manual` |

---

### `aitra_cost_per_million_tokens_usd`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** US dollars per million output tokens (energy cost only). Derived: `J/token × $/kWh ÷ 3,600 × 1,000,000`.

| Label | Description |
|---|---|
| `namespace` | Kubernetes namespace |
| `workload` | Workload type |
| `model` | Model name |
| `hardware` | GPU tier |
| `cost_source` | `openei` or `manual` |

---

### `aitra_namespace_energy_joules_total`

**Type:** Counter  
**Source:** Aggregation service  
**Description:** Cumulative energy consumed by a namespace in joules (raw, before PUE adjustment).

| Label | Description |
|---|---|
| `namespace` | Kubernetes namespace |
| `attribution_method` | `direct` or `proportional` |

---

### `aitra_namespace_tokens_total`

**Type:** Counter  
**Source:** Aggregation service  
**Description:** Cumulative output tokens generated by a namespace.

| Label | Description |
|---|---|
| `namespace` | Kubernetes namespace |

---

### `aitra_cluster_power_watts_total`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Total cluster GPU power draw in watts, summed across all nodes and devices.

| Label | Description |
|---|---|
| `cluster` | Cluster name |

---

### `aitra_idle_power_watts`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** GPU power draw in watts for a node when no inference requests are running.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |

---

### `aitra_idle_time_ratio`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Fraction of the last hour during which the node had no active inference requests. Range 0.0–1.0.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |

---

### `aitra_grid_intensity_gco2_kwh`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Current grid carbon intensity in gCO₂/kWh for the cluster's configured grid zone.

| Label | Description |
|---|---|
| `zone` | ElectricityMaps zone identifier (e.g. `SG`) |
| `source` | `electricitymaps`, `watttime`, `last_known`, or `manual` |

---

### `aitra_calibration_reference_j_per_token`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** The calibration baseline J/token for this model × hardware combination. Compare against `aitra_j_per_token` to compute efficiency delta.

| Label | Description |
|---|---|
| `model` | Model name |
| `hardware` | GPU tier |
| `calibration_tier` | `aitra_benchmark`, `reference`, or `self_calibrated` |

## Host energy metrics (#82)

Host energy is everything on the node that is **not** the accelerator: CPU
package, DRAM, and — depending on the provider — the wider board (NICs, storage,
fans, PSU losses). It is measured per node by a host-energy provider (`rapl` on
x86, `grace-hwmon` on the Grace Superchip, and the experimental opt-in
`grace-spark-hwmon` on GB10 / DGX Spark via a community driver); see the
[host energy guide](../guides/host-energy.md). The `provider` label distinguishes
these paths so experimental-path data is segregable in Prometheus.

> **Absent is not zero.** Every metric in this section is **omitted entirely** on
> nodes without host telemetry — it is never emitted as `0`. A zero would
> understate whole-node efficiency and make an *unmeasured* node look **more
> efficient** than a measured one, so the two would be incomparable and the more
> honest node would look worse. On a stock GB10 / DGX Spark, which exposes no host
> power telemetry, these metrics do not exist — unless the operator has opted into
> the experimental `grace-spark-hwmon` community-driver path.

### `aitra_host_energy_joules_total`

**Type:** Counter  
**Source:** Aggregation service  
**Description:** Cumulative host (non-accelerator) energy consumed, in joules. **Absent** on nodes without host telemetry.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `provider` | Host energy provider: `rapl`, `grace-hwmon`, `grace-spark-hwmon` (experimental) |
| `domain` | Energy domain (`all` for the summed node total) |

---

### `aitra_host_power_watts`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Current host (non-accelerator) power draw in watts. **Absent** on nodes without host telemetry.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `provider` | Host energy provider: `rapl`, `grace-hwmon`, `grace-spark-hwmon` (experimental) |

---

### `aitra_system_j_per_token`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Total system energy per output token: `(gpu_energy + host_energy) / output_tokens`. The whole-node counterpart to [`aitra_j_per_token`](#aitra_j_per_token). **Absent** on nodes without host telemetry — never falls back to the GPU-only figure.

| Label | Description |
|---|---|
| `namespace` | Kubernetes namespace |
| `workload` | Workload type: `chat`, `code`, `reasoning`, `batch`, `unknown` |
| `model` | Model name |
| `hardware` | GPU tier from node label |
| `precision` | Precision from pod annotation |
| `calibration_tier` | `aitra_benchmark`, `reference`, `self_calibrated`, `uncalibrated` |
| `attribution_method` | `direct` or `proportional` |
| `host_provider` | Host energy provider that supplied the host term |

---

### `aitra_host_energy_fraction`

**Type:** Gauge  
**Source:** Aggregation service  
**Description:** Fraction of a workload's energy consumed **outside** the accelerator: `host_energy / (gpu_energy + host_energy)`. A serving deployment burning a large fraction outside the GPU is likely misconfigured — a signal that is otherwise invisible. **Absent** on nodes without host telemetry.

| Label | Description |
|---|---|
| `node` | Kubernetes node name |
| `model` | Model name |
| `namespace` | Kubernetes namespace |

---

## Example PromQL queries

**Efficiency delta vs calibration baseline:**
```promql
(aitra_j_per_token - on(model, hardware) aitra_calibration_reference_j_per_token)
  / on(model, hardware) aitra_calibration_reference_j_per_token
```

**Cluster idle cost per hour (USD):**
```promql
sum(aitra_idle_power_watts) * 0.001 * on() aitra_grid_intensity_gco2_kwh
```

**Namespace monthly token cost:**
```promql
increase(aitra_namespace_tokens_total[30d])
  * on(namespace) aitra_cost_per_million_tokens_usd / 1e6
```

**Nodes with high idle ratio:**
```promql
aitra_idle_time_ratio > 0.5
```
